# Froggy on RAG v1.3.0 — Corpus REST API, HTTP request logging, MCP stack removed

**Release tag:** `v1.3.0` (set when publishing)  
**Package version:** 1.3.0 (bump with `npm run version:minor` before tag/build)  
**Type:** Minor release (integration surface change + documentation)

**Baseline:** Changes since **v1.2.5** only. For prior context, see **`release-notes/v1.2.5.md`**.

---

## Summary

This release completes the shift from an MCP-first integration model to **plain HTTP**: a dedicated **RAG / corpus REST server** for admin, search, and ingestion, alongside the existing **OpenAI- and Ollama-compatible LLM passthrough** listeners. The stdio MCP server, CLI entrypoint, and npm `bin` have been **removed**. The npm package is renamed to **`froggy-on-rag`** to match the repository and product positioning.

## Highlights

### Corpus REST server (replaces MCP tools)

- **`src/main/services/rag-rest-server.js`** — Express app with `/health`, `/status`, and admin routes mounted at **`/admin`** and **`/store`** (same handlers on both prefixes).
- **`src/main/services/rag-rest/`** — Admin handlers for stats, ingest, documents, chunks, and corpus search, with **namespace-aware** resolution (`namespace-scope`, `corpus-namespace-query`).
- **Typical admin routes** (under `/admin` or `/store`): `GET /stats`, `POST /ingest/file`, `POST /ingest/directory`, `GET /documents`, `GET /documents/:id`, `GET /documents/:id/chunks`, `GET /chunks/:chunkId`, `POST /corpus-search`.

### HTTP request logging

- **`src/main/services/http-request-log.js`** — Persists request metadata for debugging and auditing; log file defaults to **`http-request-log.json`** under the app root.
- **Migration:** If the new file is empty, entries are read from the legacy **`mcp-request-log.json`** so existing logs are not lost on upgrade.

### LLM passthrough (unchanged role, small fixes)

- Inbound OpenAI/Ollama listeners remain in **`passthrough-inbound-server.js`** with incremental hardening; **`llm-passthrough.js`** retains outbound sanitization from v1.2.5.

### Desktop app & IPC

- **`main.js`** wires **`RagRestServer`** next to **`RAGService`** and passthrough inbound service.
- **`ipc-handlers.js`** / **`preload.js`** — Request log access and service readiness align with REST + passthrough (no MCP bridge).

### Documentation & packaging

- **`README.md`** — Rewritten around **passthrough + dual API shapes** and the local desktop workflow; adds **screenshots** under **`images/screenshots/`** (server status, vector store, ingestion, settings, in-app tests).
- **`package.json`** — `name` → `froggy-on-rag`, `description` updated, **`bin`** and **`mcp` / `mcp-stdio`** scripts removed; keywords emphasize **llm** instead of **mcp**.
- **`scripts/create-dist.js`** — Adjusted for the current build layout.

### Removed (breaking for MCP consumers)

- **`src/cli/mcp-cli.js`**, **`src/main/services/mcp-service.js`**, MCP JSON-RPC handler, tool registry, MCP routes, **`mcp-request-log.js`**, **`scripts/mcp-example-requests.http`**, and related MCP-only assets.

### Tests

- **`test/llm-passthrough.test.js`** — Expanded coverage for passthrough behavior.
- **`test/passthrough-inbound-server.test.js`** — Covers inbound server behavior.

## Upgrade notes

- **If you integrated via MCP (stdio / Cursor MCP):** that path is gone. Use **corpus REST** on the app’s REST port for search/ingest/admin, and **LLM passthrough** ports for chat completions shaped like OpenAI or Ollama. Update any automation that invoked `npm run mcp` or the `froggy-rag-mcp` binary.
- **If you depended on the old package name:** install and reference **`froggy-on-rag`**; update CI and lockfiles accordingly.
- **Request logs:** new writes go to **`http-request-log.json`**; old **`mcp-request-log.json`** is still read once for continuity.
- **Passthrough:** v1.2.5 behavior for stripping **`froggy`** sections from upstream bodies is unchanged.
