# Froggy on RAG v1.4.0 ‚Äî Always-inject standard context, streaming passthrough, namespace VectorStore pool

**Release tag:** `v1.4.0` (set when publishing)  
**Package version:** 1.4.0 (bump with `npm run version:minor` before tag/build)  
**Type:** Minor release (retrieval feature, passthrough capability, performance)

**Baseline:** Changes since **v1.2.7** only. For prior context, see **`release-notes/v1.3.0.md`** (drafted for content that ultimately shipped across **v1.2.6** / **v1.2.7**).

---

## Summary

This release adds two user-visible capabilities and one infrastructure layer behind them. **Always-inject** lets you flag specific files or folders so every chunk from those sources is prepended to every search as **standard context** ‚Äî separate from top-K retrieval and labeled distinctly for the model. **Streaming passthrough** lifts the previous **`STREAM_NOT_SUPPORTED`** restriction on the inbound listeners: clients can now set **`stream: true`** on **`/api/chat`** (Ollama NDJSON) and **`/v1/chat/completions`** (OpenAI SSE) and Froggy pipes the upstream byte stream through unchanged, with proper client-abort propagation. Behind both of those, a new **`VectorStorePool`** keeps reference-counted, idle-timeout-cached **`VectorStore`** handles open for non-active namespaces so cross-namespace REST / passthrough traffic no longer pays SQLite cold-start cost per request.

## Highlights

### Always-inject standard context

- **`src/main/services/rag-rest/always-inject.js`** ‚Äî new module:
  - **`collectAlwaysInjectFilePaths(namespaceSettings)`** ‚Äî resolves files plus expanded supported files from folders flagged `alwaysInject === true && active !== false`.
  - **`collectAlwaysInjectHits(ragService, namespace, vectorStore)`** ‚Äî returns every chunk for those files in the standard search-hit shape, tagged with **`metadata.alwaysInject = true`**, **`algorithm: 'always-inject'`**, and **`score / similarity = 1`**.
  - **`readNamespaceSettings(ragService, namespace)`** ‚Äî reads from active in-memory settings when the namespace matches **`ragService.dataDir`**, otherwise loads **`<dataDir>/namespace.json`**.
- **`src/main/services/rag-service.js`**:
  - **`updateFileAlwaysInject(filePath, alwaysInject)`** and **`updateDirectoryAlwaysInject(dirPath, alwaysInject)`** persist the flag on file / folder entries.
- **`src/main/services/rag-rest/admin-routes.js`**, **`corpus-namespace-query.js`**, **`namespace-scope.js`** ‚Äî corpus search routes incorporate always-inject hits alongside top-K results.
- **`src/main/services/llm-passthrough.js`** ‚Äî **`formatSearchHitsForContext`** now groups always-inject hits (per namespace) under a **"Standard context (always injected)"** section ahead of regular hits, with each chunk tagged **`(always-inject)`** so the model can distinguish standard context from retrieval results. Always-inject chunks **do not** count toward **`topK`**.
- **UI (`src/renderer/app.js`, `index.html`)**:
  - New **Always Inject** column / checkbox on the files table and the folders table.
  - Tooltip: *"Prepend this file's chunks as standard context on every search (does not count toward top-K)."*
  - **`window.updateFileAlwaysInject`** / **`window.updateDirectoryAlwaysInject`** wired through **`preload.js`** and **`ipc-handlers.js`**.

### Streaming inbound passthrough (Ollama NDJSON + OpenAI SSE)

- **`src/main/services/llm-passthrough.js`**:
  - **`completeChatProxy()`** now returns a discriminated union: `{ streaming: false, upstreamJson, ‚Ä¶ }` or `{ streaming: true, upstreamWebStream, upstreamContentType, ‚Ä¶ }`.
  - Removed the **`STREAM_NOT_SUPPORTED`** error path; **`stream: true`** on the inbound body is now honored.
  - **`postJsonStream(url, body, headers, timeoutMs, abortSignal)`** ‚Äî new helper that POSTs JSON, clears the connection-phase timeout once headers arrive (so long streams aren't cut off), and returns the upstream **Web `ReadableStream`** plus its **`Content-Type`**. Abort is wired in two stages: pre-headers via the request **`AbortController`**, post-headers via **`res.body.cancel()`** when the inbound client disconnects.
  - **`parseUpstreamErrorPayload(text, status)`** ‚Äî extracted helper that pulls **`error.message`** / **`error`** / **`message`** out of upstream JSON error bodies and falls back to `HTTP <status>`. Shared by **`postJson`** and **`postJsonStream`**.
- **`src/main/services/passthrough-inbound-server.js`**:
  - **`pipeUpstreamWebStreamToResponse(req, res, webStream, { contentType })`** ‚Äî new helper that converts the upstream Web stream via **`Readable.fromWeb`**, sets `Content-Type` (defaults to **`application/x-ndjson`** for Ollama / **`text/event-stream; charset=utf-8`** for OpenAI), and forwards the bytes unchanged with proper teardown on client `close` / `aborted`.
  - Both **`/api/chat`** and **`/v1/chat/completions`** check **`out.streaming`** from **`completeChatProxy`** and either pipe the stream or fall through to the existing JSON response path.
  - Stream cancellation: when the inbound client disconnects mid-stream, the upstream **`AbortController`** is signaled and **`webStream.cancel()`** is called so upstream stops generating.

### VectorStore pool for non-active namespaces

- **`src/main/services/vector-store-pool.js`** ‚Äî new **`VectorStorePool`** class:
  - For the **active** namespace, returns **`ragService.vectorStore`** directly (no caching; **RAGService** owns the lifecycle).
  - For any **other** namespace, opens a **`VectorStore`** on first **`acquire(namespace)`** and keeps it resident.
  - Reference-counted: concurrent acquires for the same namespace share one **better-sqlite3** handle.
  - When **`refCount`** drops to zero, schedules a `setTimeout`-based close after **`idleTtlMs`** (default **5 minutes**); a re-acquire before the timer fires cancels the close. The idle timer is **`unref()`**'d so it never blocks app shutdown.
  - **`closeNow(namespace)`** / **`dispose()`** for forced teardown (e.g. namespace switch tearing down the parent **RAGService**).
  - **`getStats()`** ‚Äî diagnostic snapshot of pooled namespaces and refcounts.
- **`src/main/services/rag-service.js`**:
  - Constructs **`this.vectorStorePool = new VectorStorePool(this, { idleTtlMs: this._resolvePoolIdleTtlMs(settings) })`**.
  - Settings reload calls **`vectorStorePool.setIdleTtlMs(...)`** so changes apply on next release.
  - Disposed in the service teardown path.
- **`src/settings-files.js`** ‚Äî new default:
  - **`corpusStoreIdleTimeoutMs: 5 * 60 * 1000`** ‚Äî idle TTL for resident pooled handles.

## Upgrade notes

- **Streaming clients:** Inbound passthrough now honors **`stream: true`**. Existing clients that send `stream: false` (or omit it) are unchanged. If you previously caught the **`STREAM_NOT_SUPPORTED`** error to fall back to non-streaming, that error path is gone ‚Äî set `stream: false` explicitly to keep JSON behavior.
- **Direct consumers of `completeChatProxy`:** The return type is now a discriminated union. Update call sites to branch on **`out.streaming`** before reading **`out.upstreamJson`**.
- **Always-inject:** No-op for existing corpora ‚Äî the **`alwaysInject`** flag defaults to false. To use it, tick **Always Inject** on a file or folder row and re-run a search; the chunks appear in a **"Standard context (always injected)"** block ahead of top-K hits and do not consume top-K slots. Make sure the file or folder is **active** and **ingested**.
- **VectorStore pool:** Memory and file-handle usage may grow when multiple non-active namespaces are queried in close succession (one resident handle per namespace until the idle timer fires). Tune **`corpusStoreIdleTimeoutMs`** in **`settings.json`** if your traffic pattern needs faster reclamation; setting it to a non-positive value closes immediately on release.
- **Re-ingestion:** Not required.
- **Tests:** Run **`npm test`** after pulling.
