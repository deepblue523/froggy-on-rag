# Froggy RAG MCP v1.0.17 — Electron-aligned native rebuilds and app version source

**Release tag:** `v1.0.17`  
**Package version:** 1.0.17  
**Type:** Maintenance (install / packaging reliability)

---

## Summary

This release makes **postinstall** rebuild **native addons against the same Electron version** the app ships with, including **`sharp`** alongside **`better-sqlite3`**, so runtime ABI mismatches are less likely after `npm install` or dependency bumps. Rebuild **skips cleanly** when `electron` is not present (for example **`npm install --omit=dev`** for CLI-only workflows). The desktop shell now reports version from **`app.getVersion()`** (Electron metadata) everywhere it surfaces—window title, splash query, tray tooltip, and the **`get-app-version`** IPC—so packaged builds stay consistent with the built app version. **Windows** packaging config drops unused signing placeholders in favor of **`forceCodeSigning: false`**.

## Highlights

- **`scripts/electron-rebuild-native.cjs`** — runs `@electron/rebuild` with **`-v` set to the installed `electron` package version**, rebuilding **`better-sqlite3`** and **`sharp`**; on Windows, defaults **`npm_config_msvs_version`** to **2022** when unset.
- **Skip when Electron is missing** — if `node_modules/electron` is absent, postinstall logs once and exits **0** (supports dev-dependencies omitted for MCP-only use).
- **`package.json` `postinstall`** — now invokes the script above instead of an inline `electron-rebuild` line.
- **Version display** — tray, splash, main window title, and **`get-app-version`** use **`app.getVersion()`** instead of reading **`package.json`** at startup.
- **Diagnostics** — if service init fails with a **`better_sqlite3`**-style error, the console suggests **`npm install`** (postinstall) or an explicit **`npx electron-rebuild`** for both native modules.
- **Docs** — README clarifies that postinstall targets the **installed Electron runtime**, not system Node.

## Upgrade notes

- After pulling this version, run **`npm install`** once so postinstall can rebuild natives for your current Electron tree.
- **CLI-only / no devDependencies:** if you omit dev deps and never install Electron locally, the new postinstall **does not** fail; if you later run the full Electron app from that tree, install dev dependencies and run **`npm install`** again so native modules match Electron.
